# 自傳（800 字內）

我是一名以資安為核心的全端工程師，工作重點在把「安全」與「可交付」結合：從需求分析、系統設計、開發測試到雲端部署，讓產品不只可用，也能在真實攻擊與營運壓力下維持韌性。過去 3 年，我在高強度的專案環境中持續累積工程能力，建立 5 套自動化安全檢測腳本，協助團隊降低資安事故率 30%，並開發 3 個開源 DevSecOps 工具，在 GitHub 累積超過 200 次下載。我養成以數據與流程改善問題的習慣，例如導入自動化檢測、強化 CI/CD、建立安全基線與稽核紀錄，降低事故風險與維運成本。

在社群貢獻方面，我過去 2 年完成 10 篇技術文章，參與 3 場技術分享會，觸及超過 300 名工程師與學生。我的開源專案獲得國際開發者的關注，其中一個安全開發工具包被 15 個專案採用，展現了技術成果的國際影響力。

我選擇申請 Software Development MSc，是因為我希望把實務經驗上升為更完整的軟體工程能力：包含架構取捨、可靠性設計、可測試性、性能優化與團隊協作流程，並學會以更成熟的方法將技術轉化為可被採用的產品與服務。

在求學與工作並行的過程中，我一直面對資源與時間的限制，但也因此更重視「把努力變成可驗證成果」。我相信明緯獎學金的支持，能讓我以更穩定的節奏完成海外進修與成果產出，並在未來以工程專長投入公共利益。我計畫在完成學業後，透過開源工具、教學素材與工作坊，每年幫助至少 500 名工程師與學生提升安全開發能力，讓技術真正成為改善社會的工具，對齊 SDGs 優質教育與和平正義的目標。
